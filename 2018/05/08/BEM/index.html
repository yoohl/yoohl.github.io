BEM (Block, Element, Modifier)

웹 개발에 대한 컴포넌트 기반 접근 방식이다.
이 것은 유저 인터페이스를 독립적인 블록으로 나누는것에 기반했다.

Block

기능적으로 독립적인 페이지 컴포넌트 입니다. 자바스크립트, 템플릿, CSS 스타일 등을 담고 있으며, 독립된 블럭 요소는 재사용이 가능합니다. 블럭은 블럭 요소 안에 위치할 수 있습니다. 예로, head 안에는 로고, 검색폼, 메뉴가 포함되어 있습니다. 또한, 이러한 컴포넌트들은 블럭 요소 안에서 자유롭게 배치 되어야 합니다.

재사용 할 수 있도록 기능적으로 독립된 페이지, html 에서 블럭은 class 속성으로 표현 된다.
특징: 블럭네임은 상태가 아니라 (어떻게 생겼니? - 빨개 or 크다), 그것의 목적을 묘사한다(이게 뭐야? - 메뉴 or 버튼).

Element

블럭의 구성요소들은 블럭 밖에서는 사용을 할 수 없습니다. 예로, 메뉴라는 블럭에서 각각의 탭메뉴들은 독립적으로 다른 곳에서는 작동을 하지 않습니다. 이러한 요소를 엘리먼트라고 부릅니다.

별도로 사용할 수 없는 블럭의 복합부분
특징: element 의 전체 이름의 구조는 block-name__element-name, 더블 언더스코어__ 로 블럭네임과 구분 한다.

Modifier

블럭과 엘리먼트의 외관과 기능을 결정하는 BEM 독립체이며 필수 보다는 옵션으로 사용합니다. BEM 독립체는 blocks, elements, modifiers을 부르는 단어 입니다. Modifier 의 예로는, 상단위 위치한 탭메뉴를 하단이나 측면에 위치시킬 때는 똑같은 기능을 하지만 디자인이 수정된 것을 말합니다.
