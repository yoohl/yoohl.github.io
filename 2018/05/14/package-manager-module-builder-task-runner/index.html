패키지 매니저란?
패키지를 설치하기 위한 다운로드, 설치, 빌드, 테스트, 업그레이드, 의존성 관리 등의
복잡성을 낮추고 쉽게 패키지를 이용하도록 돕는 매니저

npm(Node Package Manager)?

node.js 에서 사용하는 모듈들을 패키지로 만들어 npm을 통하여 관리하고 배포하고 있다
다른사람이 잘 만들어 놓은 모듈을을 npm을 통해 설치하여 사용 가능하다는 점.
이 모듈이 사용하고 있는 다른 모듈의 의존성 또한 자동으로 해결 (npm install)

node.js 설치하면 자동으로 설치 됨 http://www.nodejs.org

패키지 설치

패키지 관리는 package.json이라는 파일로 관리하는데, 이 파일을 생성하는 명령어

package.json -> npm 설정 관리

Module Bundler: Webpack

모듈? 모듈은 변수의 스코프를 관리하는 기능을 한다
모듈을 사용하는 방법이 몇가지 있는데
-es6 import
-자바스크립트 파일/모듈 로더인 requireJS를 사용
-모듈 번들러인 Webpack이나 Browserify 등등을 사용

번들러? 여러가지 파일을 모아 하나로 만드는 번들링의 기능을 해주는 것
이 모듈을 하나로 합쳐 http리퀘스트 횟수를 줄여 성능 향상

이 중 웹팩을 사용하는 방법을 알아보자

글로벌 웹팩 설치

웹팩 babel 설치
-babel-core: require->import
-babel-preset-latest: ES6->ES5
-babel-loader: 웹팩과 바벨을 연동해서 사용하기 위한 로더

webpack.config.js -> 웹팩의 기본 설정 관리
