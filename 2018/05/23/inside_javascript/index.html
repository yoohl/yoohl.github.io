3.자바스크립트 데이터타입과 연산자

참조타입 객체

‘이름(KEY) : 값(VALUE)’ 의 형태로 프로퍼티들을 저장하는 컨테이너,
기본타입은 하나의 값만을 가지는 데 비해, 참조 타입인 객체는 여러 개의 프로퍼티들을 포함할 수 있음
객체의 프로퍼티는 함수로 포함할 수 있으며, 이 프로퍼티를 메서드라고 부른다.

객체 생성 방식

  1. Object() 생성자 함수 이용

  2. 객체 리터럴 방식

  3. 생성자 함수 이용

객체 프로퍼티 읽기,쓰기,갱신,삭제

  • 대괄호 ([‘문자열’]), 마침표(.) 표기법
    보통 마침표를 이용해서 프로퍼티에 접근하지만, 대괄호 표기법만을 사용해야 하는 경우가 있다
    ‘full-name’ 과 같이 ‘-‘ 연산자가 있는 표현식이다
    이런 경우 반드시 foo[‘full-name’] 으로 접근, 마침표 표기법을 사용할 경우 엉뚱하게 NaN이 출력
    삭제는 delete foo.name
    객체 프로퍼티를 삭제할 뿐, 객체 자체를 삭제하진 못한다.

for in문과 객체 프로퍼티 출력

for in문을 사용하면 객체에 포함된 모든 프로퍼티에 대해 루프를 수행 할 수 있다.

참조타입의 특성

객체의 모든 연산이 실제 값이 아닌 참조값으로 처리 된다

참조에 의한 함수 호출 방식

함수를 호출 할 때,
인자로 기본 타입의 값을 넘길 경우, 호출된 함수의 매개변수로 복사된값이 전달 -> 실제 변수의 값 변경되지 않음
인자로 참조 타입의 값을 넘길 경우, 프로퍼티 값이 아닌 참조값이 그대로 함수 내부로 전달 -> 실제 값 변경 가능

프로토타입

자바스크립트의 모든 객체는 자신의 부모역할을 하는 객체와 연결 되있다.

배열

일반적으로 리터럴로 생성, 또는 var foo = new Array(배열개수);
배열은 값을 순차적으로 넣을 필요없이 아무 인덱스 위치에나 값을 동적으로 추가 가능
모든 배열은 length 프로퍼티가 있다 이 것은 배열의 원소 개수를 나타내지만, 실제 배열 원소 개수와 일치하는 것은 아니다.
자바스크립트는 배열에서 사용 가능한 다양한 표준 메서드를 제공하고 있다.
이런 메서드는 length 프로퍼티를 기반으로 동작하고 있다. 예를 들어 push()
배열도 자바스크립트 객체이므로, 인덱스가 숫자인 배열 원소 이외에도 객체처럼 동적 프로퍼티를 추가 할 수 있다. 이 떄 length 값은 변화 안함

배열과 객체

객체 -> Object.prototype
배열 -> Array.prototype -> Object.prototype

배열요소 삭제

객체와 동일하게 delete 를 사용할 수 있으나 해당 값을 undefined로 설정할 뿐 원소 자체를 삭제하지 않는다
보통 splice() 배열 메서드를 사용한다.
splice(start(배열 시작위치), deleteCount(시작위치부터 삭제할 요소 수), item…(삭제할 위치에 추가할 요소))

유사배열객체

length 프로포티는 배열의 동작에 있어 중요한 프로퍼티이다.
일반 객체에 length 프로퍼치를 가진 객체를 유사배열객체라 부른다.
apply() 메서드를 사용하면 객체지만 표준 배열 메서드를 활용하는 것이 가능하다.

기본타입과 표준메서드

기본타입의 값들에 대해 객체 형태로 메서드로 호출할 수 있다

4.함수와 프로토타입 체이닝

##함수 정의
-함수선언문
-함수표현식
-Function() 생성자 함수

함수 선언문은 반드시 함수명이 정의되어 있어야 한다

add 변수는 함수 리터럴로 생성한 함수를 참조하는 변수이지 함수 이름이 아디라는 것에 주의.
add와 같이 함수가 할당된 변수를 함수 변수라고 부르겠다.
함수 변수 add는 함수의 참조값을 가지므로 또 다른 변수 plus 에도 그 값을 그대로 할당할 수가 있다.
add와 plus 함수 변수는 동일한 익명 함수를 참조한다.
익명함수식 vs 기명함수표현식

에러가 나는 이유는 함수 표현식에서 사용된 함수 이름이 외부 코드에서 접근 불가능하기 떄문이다
함수 표현식에서 사용된 함수 이름은 정의된 함수 내부에서 해당 함수를 재귀적으로 호출하거나(ex.팩토리얼 함수), 디버거 등에서 함수를 구분할 때 사용된다.

함수 호이스팅

함수 선언문 형태로 정의한 함수의 유효범위는 코드의 맨 처음부터 시작한다. 이것을 함수 호이스팅이라고 부른다
함수 표현식 형태로 함수를 정의하면 호이스팅이 일어나지 않아 함수가 생성 되기 전에 호출 하면 에러남
이 이유로 더글러스 크락포트는 함수 표현식만을 사용할 것을 권고 하고 있다.
이러한 함수 호이스팅이 발생하는 원인은 자바스크립트의 변수 생성과 초기화의 작업이 분리되서 진행되기 때문이다.

함수도 객체다

함수도 일반 객체처럼 취급될 수 있다. ‘일급객체’
자바스크립트 함수의 기능은 다른 언어와 비슷하게 입력한 값을 받아 처리한 다음 그 결과를 반화하는 구조다.
하지만 자바스크립트 함수는 이런 기본적인 기능 외에도 함수가 일급객체이며 일반 객체처럼 값으로 취급된다는 것을 이해해야 한다.
-변수나 객체, 배열 등에 값으로도 저장할 수 있으며,
-다른 함수의 인자로 전달 하거나
-함수의 리턴값으로도 사용 가능

함수 객체의 기본 프로퍼티

length 프로퍼티 - 인자 개수
prototype 프로퍼티 - 함수 객체의 prototype과, 모든 객체의 부모를 나타내는 내부 [[Prototype]] 혼동하지 말 것
prototype <-> construct
함수 객체와 프로토타입 객체는 서로 밀접하게 연결돼 있다

함수의 다양한 형태

콜백함수

익명함수의 대표적인 용도가 콜백 함수
단지 함수를 등록하기만 하고, 어떤 이벤트가 발생했거나 특정 시점에 도달 했을 때 시스템에서 호출되는 함수
또한 특정 함수의 인자로 넘겨서 코드 내부에서 호출되는 함수
대표적인 예가 이벤트 핸들러 처리
이벤트 -(이벤트 발생)-> 이벤트 핸들러 -(함수 호출)-> 콜백 함수

기본적으로 자바스크립트는 변수를 선언할 경우 전역 유효 범위를 가진다
그러나 함수 내부에서 정의된 매개변수와 변수들은 함수 코드 내부에서만 유효하다.(var 문을 사용하지 않으면 함수 내부라도 전역 유효범위 가짐)
-> 함수 외부 코드에서 함수 내부 변수에 엑세스 불가
따라서 라이브러리 코드를 즉시 실행 함수 내부에 정의하면 함수 외부에서 접근 불가, 전역 네임스페이스를 더럽히지 않음

내부함수

클로저를 생성하거나 부모 함수 코드에서 외부에서의 접근을 막고 독립적인 헬퍼 함수를 구현하는 용도
내부 함수는 자신을 둘러싼 외부 함수의 변수에 접근 가능하다
내부 함수는 일반적으로 자신이 정의된 부모 함수 내부에서만 호출이 가능하다

함수를 리턴하는 함수

자바스크립트에서 함수는 일급 객체이므로 일반 값 처럼 함수 자체를 리턴할 수도 있다.
함수를 호출함과 동시에 다른 함수로 바꾸거나, 자기 자신을 재정의 하는 함수를 구현할 수 있다.

함수 호출과 this

arguments 객체

함수를 호출할 때 넘긴 인자들이 배열 형태로 저장된 객체를 의미
이 객체는 실제 배열이 아닌 유사배열 객체라는 점이 특이사항

호출패턴과 this 바인딩

-메서드(객체 프로퍼티가 함수) 내부 코드에서 사용된 this는 해당 메서드를 호출한 객체로 바인딩
-함수를 호출할 때는 전역 객체 바인딩
-내부 함수를 호출 했을 경우 this는 전역에 바인딩하기 때문에 부모 메서드의 this를 내부함수가 접슨 가능한 다른변수에 저장, var that = this;
-생성자 함수를 호출할 때 this 바인딩 ? 새로 생성되는 빈 객체에 바인딩

-객체 리터럴 방식과 생성자 함수 방식의 차이
프로토타입 객체에 있음을 알 수 있다.
객체 리터럴 방식의 경우 자신의 프로토타입 객체가 Object로, 생성자 함수 방식의 경우 생성자.prototype

-call 과 apply 메서드를 이용한 명시적 this 바인딩
this를 특정 객체에 명시적으로 바인딩 시키기

함수리턴

일반 함수나 메서드는 리턴값을 지정하지 않을 경우, undefuned 값 리턴
생성자 함수에서 리턴값을 지정하지 않을 경우 생성된 객체가 리턴

프로토타입 체이닝

함수 객체의 prototype 프로퍼티와 객체의 숨은 프로퍼티인 [[Property]] 링크를 구분
자신을 생성한 생성자 함수의 prototype 프로퍼티가 가리키는 프로토타입 객체를 자신의 부모 객체로 설정하는 [[Property]] 링크로 연결한다.

5.실행 컨텍스트와 클로저

자바스크립트의 유효 범위는 블록 유효범위가 아닌 함수,
따라서 함수에서 사용하는 모든 변수는 함수 첫 부분에 선언. 함수 밖은 상관없다.
but 내부 함수에서 자신을 포함하고 있는 외부 함수의 매개변수와 변수들을 접근 할 수 있음 -> 클로저
외부 함수보다 내부 함수가 더 오래 유지 될 때?

##클로저
이미 생명주기가 끝난 외부 함수의 변수를 참조하는 함수를 클로저라고 한다.
