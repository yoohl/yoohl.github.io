Model-View-Controller (MVC) in JavaScript

2018-05-02
#js #MVC #스터디
study

자바스크립트는 가장 강력한 언어 중 하나이다.
넓은 범위의 프로그래밍 스타일과 테크닉을 지원 하지만, 그런 유연함은 위험을 가지고 온다.
좋은 예제를 따르지 않거나 불확실한 디자인패턴을 적용했을 때, 자바스크립트 프로젝트가 상대적으로 혼란에 빠지기 쉽다.

MVC

  • Model(모델) - 앱의 데이터가 저장되는 곳, 모델이 변경 될 때마다 디스패처를 사용하여 변경이 발생 했음을 관찰자에게 알림(예 : 오늘의 사용자 생일 일 경우 계산, 장바구니 품목의 총액, 세금 및 운송료 계산) api..

  • View(뷰) - 모델을 상호 작용에 적합한 양식 (일반적으로 사용자 인터페이스 요소)으로 렌더링합니다. 돔에 접근 하는 부분, click/onmouseover 등의 이벤트 핸들러 설정을 담당, html보기도 담당, 이벤트 디스패처를 사용하여 컨트롤러에게 알리고, 컨트롤러는 모델을 업데이트. html,css,js

  • Controller(컨트롤러) - updates Model by reacting on client’s actions. 뷰와 모델 사이의 접착제, 모델 또는 뷰에 설정된 이벤트를 처리하고 응답, 뷰를 업데이트 하는데도 사용

이벤트 디스패처 - 이벤트 디스패처는 무제한의 함수 / 메소드를 연결할 수있는 객체입니다. 마지막으로 Event 객체에서 notify 메소드를 호출하면 해당 이벤트에 첨부 된 모든 메소드가 실행됩니다. 아래 소스 코드에서 이런 일이 많이 일어납니다.
